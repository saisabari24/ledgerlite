# LedgerLite — VPS Deployment Guide

This guide deploys LedgerLite on a single Linux VPS using Docker Compose.
All services (PostgreSQL, NestJS backend, Next.js frontend, Caddy reverse proxy)
run in containers with auto-SSL, daily backups, and production security hardening.

---

## Architecture

```
Internet
  │
  ▼
┌─────────────────────────────────────────┐
│  Caddy (ports 80/443)                   │
│  Auto-TLS via Let's Encrypt             │
│  ├─ /api/*      → backend:3001         │
│  └─ /*          → frontend:3000        │
├─────────────────────────────────────────┤
│  Frontend (Next.js)      port 3000      │
│  Backend  (NestJS)       port 3001      │
├─────────────────────────────────────────┤
│  PostgreSQL 16            port 5432     │
│  (internal network only)                │
├─────────────────────────────────────────┤
│  Persistent Volumes:                    │
│  pg_data    → database files            │
│  uploads    → logos & documents         │
│  caddy_data → TLS certificates          │
└─────────────────────────────────────────┘
```

## Quick Start (5 steps, ~20 minutes)

### Step 1: SSH into your VPS

```bash
ssh root@<VPS-IP>
```

### Step 2: Clone the repository and set up

```bash
# Install git if needed
apt-get update && apt-get install -y git

# Clone the repo
git clone https://github.com/your-org/ledgerlite.git /opt/ledgerlite
cd /opt/ledgerlite/vps

# Run the setup script
bash setup.sh
```

The script will:
1. Install Docker + Docker Compose
2. Generate random `DB_PASSWORD` and `JWT_SECRET`
3. Create `.env` from `.env.production`
4. Tell you to set your domain

### Step 3: Set your domain

```bash
nano /opt/ledgerlite/vps/.env
```

Change this line:
```
DOMAIN=yourdomain.com          # ← change to your actual domain
```

Optional: verify other settings:
```
JWT_EXPIRES_IN=7d              # Token expiry
DISABLE_REGISTRATION=true      # Keep "true" for demos
```

### Step 4: Run setup again to deploy

```bash
cd /opt/ledgerlite/vps
bash setup.sh
```

This time it will:
1. Configure the firewall (UFW)
2. Build and start all Docker containers
3. Run database migrations
4. Seed demo data (3 businesses, 2 CAs, journal entries, customers, invoices)
5. Set up automated daily backups via cron

### Step 5: Verify it works

```bash
# Check all services are running
docker compose ps

# View live logs
docker compose logs -f

# Open in browser
# https://yourdomain.com
```

Login: `ca1@example.com` / `password123`

## Environment Variables

| Variable | Required | Default | Description |
|----------|----------|---------|-------------|
| `DOMAIN` | Yes | — | Your domain name (e.g., `demo.ledgerlite.com`) |
| `DB_PASSWORD` | Yes | — | PostgreSQL password (auto-generated by setup) |
| `JWT_SECRET` | Yes | — | JWT signing key (auto-generated by setup) |
| `JWT_EXPIRES_IN` | No | `7d` | Token expiry duration |
| `DISABLE_REGISTRATION` | No | `true` | Set to `false` to allow signups |
| `TZ` | No | `UTC` | Timezone (e.g., `Asia/Kolkata`) |

## Daily Operations

### View logs
```bash
cd /opt/ledgerlite/vps
docker compose logs -f              # all services
docker compose logs -f backend      # backend only
docker compose logs -f caddy        # access logs
```

### Restart services
```bash
cd /opt/ledgerlite/vps
docker compose restart              # restart all
docker compose restart backend      # restart backend only
```

### Update to latest code
```bash
cd /opt/ledgerlite
git pull origin main
cd vps
docker compose build                # rebuild with new code
docker compose up -d                # restart with new images
docker compose exec -T backend npx prisma migrate deploy  # apply any new migrations
```

### Database backup (manual)
```bash
cd /opt/ledgerlite/vps
bash backup.sh
```

### Restore from backup
```bash
# Copy backup to server, then:
cd /opt/ledgerlite/vps
gunzip -c /opt/ledgerlite/backups/ledgerlite_YYYYMMDD_HHMMSS.sql.gz | \
  docker compose exec -T postgres psql -U ledgerlite ledgerlite
docker compose restart backend
```

### Seed additional demo data
```bash
cd /opt/ledgerlite/vps
docker compose exec -T backend npx ts-node prisma/demo-seed.ts
```

## Security Features

| Feature | How It's Implemented |
|---------|---------------------|
| **HTTPS/TLS** | Caddy auto-provisions Let's Encrypt certificates, redirects HTTP→HTTPS |
| **Security headers** | Caddy sets X-Content-Type-Options, X-Frame-Options, XSS-Protection, Referrer-Policy, HSTS |
| **Rate limiting** | Backend uses `@nestjs/throttler` — 100 req/min general, 5 req/min on login |
| **Helmet** | Backend adds CSP, X-DNS-Prefetch-Control, Expect-CT, and other headers |
| **Registration disabled** | Default `DISABLE_REGISTRATION=true` prevents spam signups |
| **Logo upload validation** | Only PNG/JPEG/WebP accepted, max 2MB. Rejects non-image files |
| **Input validation** | NestJS `ValidationPipe` with `whitelist: true` + `forbidNonWhitelisted: true` |
| **Database isolation** | PostgreSQL runs on internal Docker network only — not exposed to internet |
| **Strong secrets** | `setup.sh` generates 256-bit random passwords for DB and JWT |
| **Firewall** | UFW configured: only ports 22 (SSH), 80 (HTTP), 443 (HTTPS) |
| **Daily backups** | Cron job runs at 3 AM, keeps 7 days of backups + monthly snapshots |
| **Tenant isolation** | All backend queries filter by `tenantId` — no cross-tenant access possible |
| **Atomic transactions** | All financial operations use `$transaction()` — no partial writes |

## Troubleshooting

### "Caddy can't get certificate"
- Make sure DNS A record points to the VPS IP
- Check port 80 is open: `curl -I http://<VPS-IP>`
- Check Caddy logs: `docker compose logs caddy`

### "Backend can't connect to database"
```bash
docker compose logs postgres         # check DB is healthy
docker compose logs backend          # check connection errors
docker compose restart postgres
```

### "Frontend shows API errors"
```bash
# Verify backend is running
curl http://localhost:3001/          # should return NestJS info

# Check NEXT_PUBLIC_API_URL
docker compose exec frontend printenv | grep NEXT_PUBLIC_API_URL
```

### "Port 80 already in use"
```bash
# Check what's using port 80
lsof -i :80

# Common culprit: nginx/apache running by default
systemctl stop nginx
systemctl stop apache2
systemctl disable nginx
systemctl disable apache2
```

### Complete reset
```bash
cd /opt/ledgerlite/vps
docker compose down -v               # destroys everything (caution!)
docker compose build --no-cache
docker compose up -d
docker compose exec -T backend npx prisma migrate deploy
docker compose exec -T backend npx ts-node prisma/seed.ts
docker compose exec -T backend npx ts-node prisma/demo-seed.ts
```
